FlexURL Documentation

Welcome to the FlexURL documentation hub. FlexURL is a secure, privacy-first, enterprise-grade redirection platform designed to help teams and developers create clean, branded short links with native analytics, bypassing bloated URL tracking parameters.

URL Shortening & Custom Aliases

By default, pasting a target URL inside the generator card yields a randomized Base62-encoded short link. Standard users can optionally supply a custom alias.

How to use:

  • Enter your target URL (e.g. https://google.com).
  • Check the **Custom Alias** checkbox to specify a clean path slug like summer-sale.
  • Submit to generate your redirect link.

Link Expiration & Fallback URL Policy

Set lifetime boundaries for marketing campaigns. Free accounts are limited to a maximum expiration of 15 days, while premium accounts can schedule absolute times or configure fallback redirect targets for expired clicks.

Fallback Configuration:

When editing a link on your dashboard, fill in the **Fallback URL** field. If a user clicks the short link after the expiration timestamp has passed, the router immediately redirects them to your fallback destination instead of displaying our default expired error template.

Real-Time Webhooks Event Sync

Sync redirection traffic telemetry logs instantly to your analytics pipeline, marketing stacks, or monitoring platforms.

When specified, the router posts a JSON payload to your webhook endpoint asynchronously during redirection: 

{
  "event": "link.redirect",
  "short_code": "x8a2f",
  "visitor": {
    "ip_country": "IN",
    "device_type": "mobile",
    "client_os": "Android",
    "referer": "https://t.co/"
  }
}

Device-Targeted Redirects

Configure custom routes depending on client operating systems. Specify separate destination URLs for **iOS** and **Android** users. For example, route users directly to their respective App Store or Google Play Store download pages, while other platforms default to the standard destination URL.

Passcode Protection Gate

Secure private materials, internal files, or beta releases. If a passcode is added to a short link, visitors must input the correct credentials on a clean verification gate page before the router authorizes redirection to the destination target.

Scheduled Activation Time

Prepare your marketing releases in advance. A scheduled short link will remain locked and display a real-time countdown timer until its specific activation time passes. You can also specify a custom countdown target URL to redirect visitors elsewhere until the activation hour arrives.

Custom Domains Setup Guide

To connect your custom brand domain (e.g., `links.mybrand.com` or `go.brand.io`) to FlexURL, you need to configure your DNS settings and verify the domain ownership.

1. Adding the Domain to Your Profile

Log in to the FlexURL Dashboard, navigate to the **Branded Domains** section, click **Add Custom Domain**, enter your domain name, and click Save.

2. Configure CNAME DNS Records

Log in to your DNS provider (e.g. Cloudflare, GoDaddy, Namecheap, Route 53) and add a CNAME record pointing your custom domain prefix (host) to FlexURL:

Record Type: CNAME | Host: links | Target/Value: flexurl.app

*Note for Cloudflare users: If proxying your domain (orange cloud), ensure your SSL/TLS encryption mode is set to **Flexible** or **Full (non-strict)** to authorize handshakes.*

3. Verify Domain Ownership

On the dashboard, click the **Verify DNS** button next to your domain. Our background system queries public records via Google DNS API to verify your configuration. Once verified, all new short links will automatically generate under your branded domain!

4. Domain Routing Constraints (Security & Isolation)

To protect and isolate your marketing campaigns, FlexURL enforces strict routing constraints on all custom domain links:

  • A short link created under a specific custom domain (e.g., `links.mybrand.com/xyz`) is only accessible via that selected domain or the default fallback `flexurl.app/xyz`.
  • Accessing the short code through other custom domains you own (e.g., `go.brand.io/xyz`) will return a 404 error.
  • Standard links shortened under the default domain `flexurl.app` cannot be accessed using your custom domains.

Server Administrator Configuration

A. Nginx Server Configuration (VPS Proxy)

Configure Nginx to catch all incoming custom domains and reverse proxy them directly to the FastAPI server running on port 8000: 

server {
    listen 443 ssl http2 default_server;
    server_name _;
    
    # SSL key paths pointing to the default flexurl.app certificate keys
    ssl_certificate /etc/letsencrypt/live/flexurl.app/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/flexurl.app/privkey.pem;
    
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

B. Cloudflare Integration Mode

To secure proxy traffic, set the Cloudflare SSL mode to **Full** (non-strict). Cloudflare will validate and encrypt traffic to your VPS origin by accepting your default Nginx server certificate fallback during handshakes.
